Features

Everything you need, nothing you don't

Purpose-built for ResilienceScan survey coordinators who need to go from raw data to delivered reports without fighting with software.

📄

Personalised PDF reports

One professionally-typeset, LaTeX-rendered PDF per respondent — radar charts, score tables, and narrative sections generated from their individual data.

⚡

Batch generation

Render reports for hundreds of respondents in one click. The GUI shows live progress and lets you cancel at any time.

📧

Automated email delivery

Send each report directly to the right recipient via Outlook COM (Windows) or SMTP. Re-runs skip people who already received their report.

📥

Multi-format import

Import respondent data from Excel (.xlsx, .xlsm, .xls), CSV, JSON, JSONL, ODS, or SpreadsheetML. New records are upserted without overwriting existing send history.

📦

Fully self-contained

R, Quarto, TinyTeX, and all 19 R packages are bundled inside the installer. No internet access, no post-install scripts, no dependency hunting.

✅

Built-in validation

Every generated PDF is automatically validated against the source CSV — catching rendering errors before reports reach respondents.

🔒

Secure SMTP credentials

Passwords are stored in the OS keyring (Windows Credential Manager, macOS Keychain, or Linux Secret Service) — never in plaintext config files.

🖥️

Cross-platform GUI

Native look-and-feel on Windows (Sun Valley theme), macOS, and Linux via Tkinter. No browser, no Electron, no cloud dependency.

How it works

Four steps from data to inbox

The GUI walks you through each stage. Every step can also be run independently from the command line.

1

Import

Load an Excel, CSV, or JSON file. Columns are normalised and upserted into the master data store.

2

Generate

One PDF per respondent is rendered via Quarto + R + TinyTeX. Radar charts, score tables, and narrative auto-populated.

3

Validate

Every PDF is checked against the source data. Mismatches are flagged before anything is sent.

4

Send

Reports are emailed to each respondent. Status tracked per-person so re-runs never double-send.
